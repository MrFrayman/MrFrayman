Hey. I’m **Syed Maarif R. Hussein** — based in Sharjah, studying Software Engineering at Liverpool John Moores University. I build software, ship it, break it, and try to understand *why* it broke instead of pretending it won’t happen again.

I’m mostly interested in **how software behaves in the real world**: deployments, environments, failures, and the boring-but-important stuff that keeps systems running. Web apps are just the surface area.

## Projects Worth Looking At

### **Read Up**

A Discord bot that helps users search books via Google Books and maintain a personal reading library inside a server.

**What it does**

* `/search` fetches structured book data (cover, pages, rating)
* Users can save titles to their personal library
* `/library` displays previously saved books

**Tech**

* Node.js + Discord.js (v14)
* Supabase (persistent storage)
* Google Books API
* Deployed on Railway with environment-based config

This was built to solve a real annoyance: sharing reading lists without copy-pasting links.

---

### **LocalStack Monitor** *(WIP)*

A desktop application exploring **local-first telemetry** and system observability.

**Concept**

* Qt on C++ based desktop client
* Collects local service metrics and logs
* Stores data locally (SQLite, optional encryption)
* Ships structured data to a backend for aggregation

This project is intentionally opinionated and experimental. Design decisions are documented as I go.
